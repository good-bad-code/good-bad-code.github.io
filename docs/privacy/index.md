# 개인정보처리방침

**최종 업데이트**: 2026년 6월 13일

## 1. 개요

Entry Sync("본 확장 프로그램")는 엔트리(Entry, playentry.org) 작품에서 `?!` 접두사가 붙은 변수와 리스트를 실시간으로 동기화하는 Chrome 확장 프로그램입니다.

본 개인정보처리방침은 확장 프로그램이 수집, 사용, 저장하는 데이터와 그 목적을 설명합니다.

## 2. 수집하는 데이터

### 2.1 수집 항목

| 데이터 | 설명 | 필수 여부 |
|--------|------|-----------|
| **프로젝트 URL** | 사용자가 접속 중인 Entry 작품 페이지의 전체 URL (`window.location.href`) | 필수 (동기화 식별자) |
| **변수명과 값** | Entry 작품 내 `?!` 접두사가 붙은 변수의 이름(`name`)과 값(`value`) | 필수 (동기화 대상) |
| **리스트명과 배열** | Entry 작품 내 `?!` 접두사가 붙은 리스트의 이름과 전체 데이터 배열 | 필수 (동기화 대상) |
| **타임스탬프** | 변수/리스트가 변경된 시각 | 필수 (변경 순서 보장) |

### 2.2 수집하지 않는 데이터

다음 데이터는 **수집하지 않습니다**:

- 이름, 이메일, 전화번호 등 개인 식별 정보
- 비밀번호, 인증 토큰 등 인증 정보
- IP 주소 (Supabase 서버 로그에 기록될 수 있으나 확장 프로그램이 수집/저장하지 않음)
- 위치 정보 (GPS, 지역 등)
- 건강 정보
- 금융 및 결제 정보
- 방문 기록 (브라우징 히스토리)
- 키 입력, 마우스 움직임 등 사용자 활동 로그
- 쿠키, 추적용 식별자
- 광고 관련 데이터

### 2.3 자동 수집 데이터

확장 프로그램이 **사용자의 개별 입력 없이 자동으로 감지하여 전송**하는 데이터는 오직 Entry 런타임에서 `?!` 접두사가 붙은 **변수와 리스트의 값**뿐입니다. 이는 사용자가 Entry 블록 코딩 과정에서 의도적으로 생성한 데이터입니다.

## 3. 데이터 수집 방법

```
Entry Runtime (playentry.org 페이지, MAIN world)
  ↓ inject.ts가 Variable.prototype.setValue, ListVariable 메서드를 hook
  ↓ content.ts가 window.postMessage 수신 → chrome.runtime.sendMessage
  ↓ background.ts (Service Worker)
  ↓ Supabase REST API (fetch)
  ↓ Supabase PostgreSQL DB (sync_variables, sync_lists 테이블)
```

확장 프로그램이 페이지에 삽입하는 스크립트(`inject.js`)가 Entry 런타임의 변수/리스트 변경을 감지하여 Supabase 데이터베이스로 전송합니다.

## 4. 데이터 저장 및 보관

### 4.1 저장 위치

모든 수집 데이터는 **Supabase** 클라우드 데이터베이스(PostgreSQL)에 저장됩니다.
- 서비스 제공자: Supabase Inc.
- 서버 위치: (Supabase 표준 인프라, 리전은 서비스 제공자에 따름)
- DB 인스턴스: `qjjfyuxomsuflczcgktf.supabase.co`

### 4.2 보관 기간

데이터는 사용자가 Entry 작품을 사용하는 기간 동안 유지됩니다. 명시적인 데이터 삭제 기능은 현재 제공되지 않으며, 추후 업데이트를 통해 제공될 예정입니다.

### 4.3 보안

- **전송 중 암호화**: 모든 통신은 HTTPS(TLS) 또는 WSS(WebSocket Secure)를 통해 암호화됩니다.
- **저장 중 암호화**: Supabase는 저장 데이터 암호화를 제공합니다.
- **인증**: Supabase Anon Key를 통한 제한된 접근 (RLS 정책 기반).

## 5. 데이터 사용 목적

수집된 데이터는 **오직 하나의 목적**으로만 사용됩니다:

> **Entry 작품의 `?!` 변수/리스트를 여러 탭, 창, 사용자 간 실시간으로 동기화**

이를 위해 데이터는 다음과 같이 처리됩니다:
- 현재 탭의 변경사항 → Supabase DB에 저장
- Supabase DB 변경사항 → Realtime WebSocket을 통해 다른 연결된 탭/사용자에게 브로드캐스트
- 각 탭은 수신한 변경사항을 Entry 런타임에 적용
